—— The Problem

Manual Changes Break Clusters Silently

Every oc patch, oc delete, or oc edit on a live cluster creates invisible drift from the desired state.

Security Bypass

NetworkPolicies deleted, RBAC roles escalated, resource quotas removed — all without audit trail.

Invisible Risk

No diff, no review, no rollback. The cluster looks fine until something fails in production.

Repeated Fixes

Manual changes get overwritten, forgotten, or conflict with the next deployment cycle.

Compliance Gaps

Auditors can't verify cluster state matches policy when changes bypass version control.

—— Why This Matters
70%
of unplanned outages trace back to configuration changes and human error — not hardware or software failures.
Source: Gartner — Configuration Management Best Practices
—— The Solution

Git Becomes the Enforcement Layer

ArgoCD continuously compares cluster state against Git. When drift is detected, it automatically syncs back to the desired state.
# bootstrap/argocd-config/cluster-config-application.yaml
syncPolicy:
  automated:
    selfHeal: true   # Revert manual changes
    prune: true      # Remove orphaned resources
  retry:
    limit: 5
    backoff:
      duration: 5s

—— What Gets Deployed

Four Categories of Enforced Configuration

Console Banner

Blue ConsoleNotification banner: "This cluster is managed by GitOps" with a link to the repo.

RBAC

Read-only ClusterRole (gitops-demo-viewer) bound to authenticated users. No write access.

Network Policy

Deny-all egress NetworkPolicy in the gitops-demo-secured namespace.

Resource Quota

LimitRange (max 2 CPU) and ResourceQuota in gitops-demo-quotas.

—— Getting Started

Three Commands to Demo-Ready

  • 1. Configure your GitHub org:
    ./demo/demo-script.sh --setup <your-org>
  • 2. Commit and push to your remote:
    git add -A && git commit -m "Configure org" && git push
  • 3. Run the interactive demo:
    ./demo/demo-script.sh
Cleanup: ./demo/teardown.sh removes all resources, handles stuck finalizers, and restores placeholders.
—— Takeaways

What Changes with GitOps Drift Prevention

  • 1 Git is the single source of truth. All cluster configuration is version-controlled, reviewable, and auditable.
  • 2 Drift is corrected automatically. ArgoCD selfHeal + prune reverts unauthorized changes within seconds.
  • 3 Security policies can't be bypassed. RBAC, NetworkPolicies, and ResourceQuotas are continuously enforced.
  • 4 Compliance is provable. ACM governance policies record every drift event for audit.
  • 5 Zero manual intervention required. Detection, correction, and recording happen without human involvement.
